# Contribution au SDK GoPay

Merci de votre intérêt pour le SDK GoPay.

Les contributions sont les bienvenues sous forme de :

* Correction de bugs
* Amélioration de la documentation
* Optimisations de performance
* Nouvelles fonctionnalités compatibles avec l'architecture existante

## Règles générales

* Respecter les standards PSR-12
* Utiliser des noms explicites
* Éviter les dépendances inutiles
* Préserver la compatibilité avec les versions supportées de Laravel
* Maintenir la cohérence avec l'architecture du SDK

## Procédure

### 1. Forker le dépôt

Créez un fork du dépôt officiel GoPay.

### 2. Créer une branche dédiée

```bash
git checkout -b feature/ma-fonctionnalite
```

### 3. Effectuer les modifications

Apportez vos changements en respectant les conventions du projet.

### 4. Tester le package

Vérifiez que vos modifications fonctionnent correctement avant toute soumission.

### 5. Soumettre une Pull Request

Ouvrez une Pull Request détaillant clairement :

* le problème résolu
* les modifications apportées
* les éventuels impacts

## Versions officielles

Le SDK GoPay est open-source et peut être utilisé, modifié, forké ou redistribué conformément à sa licence.

Toutefois, seules les versions publiées et distribuées par GoPay sont considérées comme des versions officielles.

Les versions modifiées, forkées ou redistribuées par des tiers ne sont ni certifiées, ni auditées, ni maintenues, ni supportées par GoPay.
